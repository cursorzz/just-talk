# JustTalk 产品与实现决策

本文档是当前版本功能行为的统一依据。后续如果产品行为发生变化，应在同一个提交中同步修改本文档。

## 产品定位

- 应用名称统一为 **JustTalk**，使用 `com.justtalk.slim` 作为 bundle identifier。
- 使用 Rust + Tauri + 系统 WebView 实现轻量、可发布的桌面应用，支持 macOS、Windows 和 Linux。
- 保留原版的核心语音输入体验，不捆绑 Python、Qt 或 Chromium。
- 桌面应用提供主窗口、全局快捷键、桌面级录音浮窗和系统托盘；关闭主窗口时应用继续驻留托盘。
- 应用图标沿用原项目；macOS Dock 图标保留合理留白，托盘图标使用可随系统主题变化的单色模板图标。

## 语音识别

- 继续使用原版接入的火山引擎豆包 SAUC 大模型流式语音识别服务：`bigmodel_nostream`。
- 录音参数与原版对齐：16 kHz、单声道、16-bit PCM，以 200 ms 音频帧持续发送。
- 建立 WebSocket 连接前允许先采集并缓存音频，避免吞掉快捷键按下后的开头内容。
- 停止时等待 150 ms 排空录音缓冲，发送 V1 协议结束帧并等待最终识别结果；最终响应兜底等待 1.5 秒。
- “正在连接”和“正在录音”在用户界面中统一显示为“正在聆听”，内部仍保留不同状态以处理连接和错误。
- 识别完成后复制结果到剪贴板；“自动粘贴”开启时，通过系统输入能力粘贴到当前前台应用。
- 识别语言、热词、自动标点、语义顺滑和压缩传输属于高级设置。
- App ID 与 Access Token 保存在操作系统的本地应用配置目录，不发送给 JustTalk 自有服务器。
- 用户填写 App ID 与 Access Token 后，可用“测试连接”发起真实服务请求验证配置。
- 调试模式开启后，调试面板显示请求参数、音频帧元数据和服务返回；Access Token 必须脱敏。

## 录音方式

- 默认使用“自由模式”：按住全局快捷键开始录音，松开立即停止。
- “常规模式”：第一次按快捷键开始录音，第二次按快捷键停止；快捷键释放不触发停止。
- 页面“开始录音”按钮可独立启动录音，再次点击停止。
- 不论通过快捷键还是页面按钮，只要触发停止或取消，桌面浮窗必须立即消失，不等待语音识别返回。
- 快速按下又松开快捷键时，迟到的连接或浮窗事件不得让浮窗重新出现。
- 快捷键修改采用捕获模式：点击“更改快捷键”后直接按新组合；注册失败或已占用时保留旧快捷键并展示错误，成功后立即写入。

## 录音浮窗

- 浮窗是跨平台、置顶、无任务栏入口的桌面级窗口。
- 浮窗尺寸固定，胶囊保持紧凑，不随状态文字改变窗口大小，以避免透明窗口出现大黑边。
- 录音过程中只显示“正在聆听”和实时波形，不显示确认或取消按钮。
- 波形根据麦克风输入音量变化，而不是播放固定动画。
- 进入停止、取消、处理或失败状态时浮窗立即隐藏。

## 录音时暂停媒体

- 高级设置提供“录音时暂停媒体”，默认开启，修改后立即生效。
- 开始录音时，只暂停当时处于播放状态且已注册系统媒体会话的播放器。
- 停止、取消、录音启动失败或识别任务异常退出时，立即尝试恢复 JustTalk 暂停的媒体。
- 只恢复由 JustTalk 记录的同一媒体会话；原本暂停的媒体不会被启动，用户已经手动恢复的媒体不会再次控制。
- 此功能不是全局静音：不控制通知声、游戏音效、通话或没有注册媒体会话的音频源。
- Windows 使用 Global System Media Transport Controls；Linux 使用标准 MPRIS；macOS 使用 MediaRemote/JXA。
- macOS 的 MediaRemote 是 Apple 私有接口。普通签名 DMG 可使用，但未来系统升级可能改变行为，并且此实现不以 Mac App Store 审核为目标。

## 设置与界面

- 主界面保持紧凑、窄窗口和较小左右留白，按“本次识别、输入方式、语音服务、高级设置”的顺序组织。
- 不为识别结果额外提供“复制”和“清空”按钮。
- 输入方式与高级设置的任意调整都立即保存并生效，不提供额外保存按钮。
- App ID 与 Access Token 只有点击“保存服务”时才持久化，避免输入中的半成品配置覆盖可用配置。
- 高级设置默认折叠，不在折叠标题中罗列内部字段。
- 调试面板仅在调试模式开启时展示。

## macOS 权限

- 完整功能要求麦克风和辅助功能两项权限全部开启；权限未齐时主功能保持门禁状态。
- 麦克风权限用于录音；辅助功能权限用于将识别结果粘贴到其他应用。
- 全局快捷键使用系统注册接口，不监听任意键盘输入，因此不要求“输入监控”权限。
- 权限页面列出每项权限、当前状态和开启入口；窗口重新获得焦点时自动复检。
- 如果辅助功能已开启但应用仍检测不到，用户应删除系统设置中的旧条目，重新添加 `/Applications/JustTalk.app` 并重启应用。
- 正式升级必须保持 bundle identifier 和签名身份稳定，否则 macOS 可能把新版本识别成另一个应用。

## 发布与构建

- Makefile 是本地开发、测试和构建的统一入口。
- macOS 生成 `.app` 和 DMG，并对完整应用执行 ad-hoc 签名；正式分发可替换为 Developer ID 签名与公证。
- GitHub Actions 支持 macOS ARM64 DMG、Windows x64 NSIS、Linux x64 AppImage/DEB 三个平台产物。
- 推送 `v*` 标签或手动触发工作流进行发布构建。
